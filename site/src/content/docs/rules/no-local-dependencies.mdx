This rule enforces that no `dependencies` are declared with local file paths, since this will likely result in errors when installing from a registry.
This includes any paths that start with `file:`, `link:`, `./`, or `../`.

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
  "dependencies": {
    "foo": "file:../foo",
    "bar": "link:../bar",
    "baz": "../baz"
  }
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
  "dependencies": {
    "foo": "workspace:^",
    "bar": "^1.2.3",
    "baz": "~0.1.2"
  }
}
```

</TabItem>
</Tabs>

### `ignorePrivate`

When enabled, `ignorePrivate` skips this check entirely.
This is useful for internal packages that won't be published to npm.
Disable if you want to enforce for all packages, regardless of privacy.

```json
{
  "package-json/no-local-dependencies": [
    "error",
    {
      "ignorePrivate": false
    }
  ]
}
```
