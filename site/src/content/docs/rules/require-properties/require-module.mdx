import { Aside, Tabs, TabItem } from '@astrojs/starlight/components';

This rule checks for the existence of the `"module"` property in a package.json, and reports a violation if it doesn't exist.

<Aside>
  The `module` property is generally not needed anymore, if you're using a
  modern bundler that supports the `exports` field. Consider using the
  [`require-exports`](/rules/require-properties/require-exports) rule instead,
  which requires that the `exports` field is present.
</Aside>

## Examples

<Tabs>
<TabItem label="❌ Incorrect">

```json
{
  "name": "thee-silver-mt-zion",
  "version": "13.0.0"
}
```

</TabItem>
<TabItem label="✅ Correct">

```json
{
  "name": "my-package",
  "version": "13.0.0",
  "module": "./index.js"
}
```

</TabItem>
</Tabs>
